Friday February 19, 2016 - Git and Push Requests

I just updated a project so now I need to quickly figure out how to move these changes to the Git server (GitHub).

My goal was to use Git and GitHub as a way to work on my projects from whatever machine I happened to be at at the time. Git is the version control system and GitHub is a remote Git server where the source code can be stored. In the beginning I thought they were one in the same. I installed Git and GitHub desktop however I'm fairly certain all I really needed was Git and a GitHub account. I tried using the GitHub GUI but that was bullshit and didn't seem to work all too well, I couldn't figure out a good workflow and get things to sync up the way I wanted them to. Currently I am using the Git bash and a GitHub repository I created.

$ cd ..
#up one directory

$ cd /target/directory
# change to target directory

$ git init
# initialize directory with Git info

$ git remote add origin http://remote.repositoryURL.git
# Sets the new remote

$ git remote -v
# Verifies the new remote URL


updating the repository after completing some work on it.

$ git add .
# Adds the files in the local repository and stages them for commit. To unstage a file, use 'git reset HEAD YOUR-FILE'.

$ git commit -m "First commit"
# Commits the tracked changes and prepares them to be pushed to a remote repository. To remove this commit and modify the file, use 'git reset --soft HEAD~1' and commit and add the file again.

$ git push origin master
# Pushes the changes in your local repository up to the remote repository you specified as the origin


Getting changes from the repository

git fetch remotename
# Fetches updates made to a remote repository

git merge remotename/branchname
# Merges updates made online with your local work

git pull remotename branchname
# Grabs online updates and merges them with your local work
